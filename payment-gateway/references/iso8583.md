# ISO8583 — Message Format, Fields & Protocols

## ISO8583:1987 vs 1993 vs 2003

| Version | Max DEs | Bitmap | Notes |
|---------|---------|--------|-------|
| 1987 | 76 | Primary only | Original |
| 1993 | 76/128 | Primary + Extended | DE64+ added |
| 2003 | 128 | Primary + Extended | Extended bitmap flag |

## MTI — Message Type Indicator

```
1st digit:  Message Class
  0xxx = Reserved (network)
  1xxx = Authorization
  2xxx = Financial
  3xxx = File Update
  4xxx = Reversal/Chargeback
  5xxx = Reconciliation
  6xxx = Administrative
  7xxx = Fee Collection
  8xxx = Network Management

2nd digit:  Message Function
  x0xx = Request
  x1xx = Request Response (advice)
  x2xx = Response
  x3xx = N/A
  x4xx = Notification
  x5xx = N/A
  x6xx = N/A
  x7xx = N/A
  x8xx = Acknowledgment
  x9xx = N/A

3rd digit:  Message Origin
  xx0x = Acquirer
  xx1x = Acquirer + Visa
  xx2x = Issuer
  xx3x = Issuer + Visa
  xx4x = Other
  xx5x = VisaDirect

4th digit:  Version
  xxx0 = ISO8583:1987
  xxx1 = ISO8583:1993
  xxx2 = ISO8583:2003
```

## Bitmap Structure

### Primary Bitmap (8 bytes = 64 DEs)
- Bit set to 1 = DE present
- DE numbers derived from bit position (1-64)
- If bit 1 is set → extended bitmap present

### Extended Bitmap (8 bytes = next 64 DEs, 65-128)
- Only present if bit 1 of primary bitmap = 1

### Bitmap Example (ASCII)
```
0123456789ABCDEF  →  hex notation
0x30 0x31 ...     →  if ASCII '0' (0x30) = 0011 0000
                     bit 6 set = DE 6 present
```

### Binary bitmap notation (for parse):
```
B'01100000' = byte 0x60 = ASCII '`' = bit 6 set → DE 6 present
```

## Data Element Definitions (DE 1-128)

### DE-001: Primary Bitmap
- **Format:** Fixed 8 bytes (binary) or 16 ASCII hex chars
- **Position:** Always first after MTI
- **Notes:** Bit 1 = extended bitmap flag; Bit 64 = DE 64 presence

### DE-002: Primary Account Number (PAN)
- **Format:** Variable, 13-19 digits (LLVAR/LLLVAR)
- **Notes:** Must pass Luhn (ISO/IEC 7812); max 19 digits
- **Validations:** No all zeros, no spaces, numeric only
- **Security:** Encrypted/transmitted via secure channel; tokenized via vault

### DE-003: Processing Code
- **Format:** Fixed 6 digits (NNXXXX)
  - Bytes 1-2: Transaction Type Code
    - `00` = Normal transaction
    - `01` = Withdrawal
    - `09` = Refund
    - `20` = Deposit
    - `30` = Account Transfer
  - Bytes 3-4: From account type
  - Bytes 5-6: To account type
- **Notes:** Used for routing decisions

### DE-004: Transaction Amount
- **Format:** Fixed 12 digits (N12), right-justified, zero-padded
- **Notes:** No decimal point; last 2 digits = minor unit (cents)
- **Example:** `000000012500` = €125.00
- **Limits:** Per network & merchant category

### DE-005: Settlement Amount
- **Format:** Fixed 12 digits (N12)

### DE-006: Cardholder Billing Amount
- **Format:** Fixed 12 digits (N12)
- **Notes:** In cardholder's billing currency

### DE-007: Transmission Date/Time
- **Format:** MMDDhhmmss (10 digits)
- **Example:** `0525103025` = May 25, 10:30:25

### DE-008: Cardholder Billing Fee Amount
- **Format:** Fixed 8 digits (N8)

### DE-009: Settlement Conversion Rate
- **Format:** Fixed 8 digits (N8)

### DE-010: Cardholder Billing Conversion Rate
- **Format:** Fixed 8 digits (N8)

### DE-011: Transmission Sequence Number
- **Format:** Fixed 6 digits (N6)
- **Notes:** Unique per terminal per day (STAN)

### DE-012: Local Transaction Time
- **Format:** hhmmss (6 digits)
- **Example:** `103025` = 10:30:25

### DE-013: Local Transaction Date
- **Format:** MMDD (4 digits)

### DE-014: Card Expiration Date
- **Format:** YYMM (4 digits)
- **Notes:** 2-digit year + 2-digit month; must be future date

### DE-015: Settlement Date
- **Format:** MMDD (4 digits)

### DE-016: Currency Conversion Date
- **Format:** MMDD (4 digits)

### DE-017: Capture Date
- **Format:** MMDD (4 digits)

### DE-018: Merchant Type (MCC)
- **Format:** Fixed 4 digits (N4)
- **Example:** `5411` = Grocery stores; `5812` = Eating places

### DE-019: Acquiring Institution Country Code
- **Format:** Fixed 3 digits (Numeric ISO 3166-1)
- **Example:** `276` = Germany; `250` = France; `826` = UK

### DE-020: Primary Extension Account Number
- **Format:** Fixed 3 digits (N3)

### DE-021: Card Acceptor Institution Country Code
- **Format:** Fixed 3 digits (Numeric ISO 3166-1)

### DE-022: POS Entry Mode
- **Format:** Fixed 3 digits (NNN)
- **Byte 1:** PAN Entry
  - `0` = No terminal
  - `1` = Manual entry
  - `2` = Magnetic stripe
  - `3` = Barcode
  - `5` = IC/Chip card
  - `6` = contactless chip
  - `7` = contactless contactless
  - `8` = credential on file
  - `9` = fallback
- **Byte 2:** PIN Entry
  - `0` = No PIN capability
  - `1` = PIN captured
  - `2` = No PIN (signature)
  - `3` = Chip & PIN

### DE-023: Card Sequence Number
- **Format:** Fixed 3 digits (N3)
- **Notes:** Sequence number for multi-card accounts (replaced cards)

### DE-024: Network International ID
- **Format:** Fixed 3 digits (N3)
- **Notes:** Used in network settlements

### DE-025: POS Condition Code
- **Format:** Fixed 2 digits (N2)
- **Values:** `00`= Normal; `01`= Partial approval; `08`= Pre-auth

### DE-026: POS PIN Capture Code
- **Format:** Fixed 2 digits (N2)
- **Values:** `0-9` = Number of PIN digits; `99` = Not captured

### DE-027: Authorization ID Response Length
- **Format:** Fixed 1 digit (N1)

### DE-028: Transaction Fee Amount
- **Format:** Fixed 10 digits with sign (N10), signed (+/-)

### DE-029: Settlement Fee Amount
- **Format:** Fixed 10 digits with sign (N10)

### DE-030: Transaction Processing Fee Amount
- **Format:** Fixed 10 digits with sign (N10)

### DE-031: Settlement Processing Fee Amount
- **Format:** Fixed 10 digits with sign (N10)

### DE-032: Acquiring Institution ID Code
- **Format:** Variable up to 11 digits (LLVAR)
- **Notes:** The Visa/MC assigned acquirer ID; used for routing

### DE-033: Forwarding Institution ID Code
- **Format:** Variable up to 11 digits (LLVAR)
- **Notes:** Network routing

### DE-034: Primary Extension Account Number Field
- **Format:** Variable up to 28 digits (LLLVAR)
- **Notes:** For secondary PANs

### DE-035: Track 2 Data
- **Format:** Variable up to 37 chars (LLLVAR), contains:
  - PAN; semicolon; exp date; service code; PVV/pin verify; cardholder name; ^ ; remainder
- **Example:** `5436031030606378=05051010300000000000`
- **Notes:** Typically encrypted at terminal level via SPDH/HPDH

### DE-036: Track 3 Data
- **Format:** Variable up to 104 chars (LLLVAR)

### DE-037: Retrieval Reference Number
- **Format:** Fixed 12 digits
- **Notes:** Unique transaction ID across settlement lifecycle

### DE-038: Authorization ID Response
- **Format:** Fixed 6 digits
- **Notes:** Approval code (for approved transactions) or rejection reason

### DE-039: Action Code / Response Code
- **Format:** Fixed 2-3 digits (varies by network)
- **Visa codes:** 00=Approved, 05=Decline, 51=Insufficient funds, 14=Invalid card
- **MC codes:** 00=Approved, 05=Do not honor, 12=Invalid transaction

### DE-040: Card Acceptor Business Code
- **Format:** Fixed 4 digits

### DE-041: Card Acceptor Terminal ID
- **Format:** Fixed 8 digits
- **Notes:** Unique per terminal; used in settlement disputes

### DE-042: Card Acceptor ID Code
- **Format:** Fixed 15 digits (LLLVAR)
- **Notes:** Merchant ID (MID)

### DE-043: Card Acceptor Name/Location
- **Format:** Fixed 40 chars
- **Layout:** City (13) + State (2) + Country (2) + Name (23)
- **Example:** `PARIS IDFR MERCHANT ONE`

### DE-044: Additional Response Data
- **Format:** Variable up to 25 chars (LLVAR)

### DE-045: Track 1 Data
- **Format:** Variable up to 76 chars (LLLVAR)
- **Example:** `B5436031030606378^DOE/JOHN^050510103000000000000000`

### DE-046: Additional Data — ISO
- **Format:** Variable (LLVAR/LLLVAR)

### DE-047: Additional Data — National
- **Format:** Variable

### DE-048: Additional Data — Private
- **Format:** Variable (LLLVAR)
- **Notes:** Network-specific, VISA/MC proprietary fields

### DE-049: Transaction Currency Code
- **Format:** Fixed 3 digits (ISO 4217)
- **Examples:** `978`=EUR, `826`=GBP, `756`=CHF, `985`=PLN

### DE-050: Settlement Currency Code
- **Format:** Fixed 3 digits

### DE-051: Cardholder Billing Currency Code
- **Format:** Fixed 3 digits

### DE-052: PIN Data
- **Format:** Fixed 8 bytes (binary)
- **Notes:** Encrypted PIN block per ISO 9564 (Format 0/1/4)

### DE-053: Security Related Control Info
- **Format:** Fixed 16 bytes (binary)

### DE-054: Additional Amounts (Binary)
- **Format:** Variable
- **Notes:** Various balance/cashback amounts

### DE-055: EMV Chip Data (ICR)
- **Format:** Variable (LLLVAR)
- **Notes:** TLV-encoded EMV data from card chip (ARQC/TC data)
- **Common Tags:**
  - `9F02` = Amount, Authorized (numeric)
  - `9F03` = Amount, Other (numeric)
  - `9F10` = Issuer Application Data (IAD)
  - `9F26` = Application Cryptogram (ARQC)
  - `9F36` = Application Transaction Counter (ATC)
  - `9F37` = Unpredictable Number (UN)
  - `95` = Terminal Verification Results (TVR)
  - `82` = Application Interchange Profile (AIP)
  - `84` = Dedicated File Name (DFN)
  - `9A` = Transaction Date (YYMMDD)
  - `9B` = Transaction Status Analysis (TSA)
  - `9C` = Transaction Type

### DE-056: Original Data Elements
- **Format:** Fixed 42 chars
- **Notes:** For stand-in / reversal messages

### DE-057: Authorization Life Cycle Code
- **Format:** Fixed 3 chars

### DE-058: Authorization Service Code
- **Format:** Fixed 3 chars

### DE-059: Original Transaction Time
- **Format:** MMDDhhmmss

### DE-060: Original Transaction Date
- **Format:** MMDD

### DE-061: Original Trace Number (STAN)
- **Format:** 6 digits

### DE-062: Card Issuer Reference Data
- **Format:** Variable (LLLVAR)

### DE-063: Settlement Service Indicator
- **Format:** Fixed 3 digits

### DE-064: Message Authentication Code (MAC)

## Variable-Length Field Encoding

| Type | Format | Max Length | Used For |
|------|--------|-----------|---------|
| LLVAR | 2-digit length + data | 99 chars | Short fields |
| LLLVAR | 3-digit length + data | 999 chars | Long fields |
| Fixed | Pad with spaces/zeros | N | Fixed fields |

## SPDH Protocol (Secure Payment Data Host)

SPDH is used between POS terminal and payment gateway for point-to-point encrypted communication.

### SPDH Message Structure
```
STX | MESSAGE_TYPE | LENGTH | DATA | CHECKSUM | ETX
02    4 chars        4 N     N       4         03
```

### SPDH Session Keys
- Terminal and host derive session keys using RSA key exchange
- Data encrypted with 3DES (or AES for newer terminals)
- MAC computed over encrypted payload

### SPDH Commands
- `0100` = Authorization Request
- `0110` = Authorization Response
- `0200` = Financial Request
- `0210` = Financial Response
- `0800` = Key Exchange Request
- `0810` = Key Exchange Response

### HPDH (Host Protocol Data Host)
- Similar to SPDH but for host-to-host
- Used for acquirer connections, terminal management systems

## Binary ISO8583 Format

Unlike ASCII ISO8583 (human-readable hex strings), binary uses actual byte values:

```
ASCII:  "0110" = 0x30 0x31 0x31 0x30 (4 bytes)
Binary: 0x01 0x10 (2 bytes, actual MTI value)
```

Field values also encoded as binary digits, not ASCII hex.

## Parsing Flow

```python
def parse_iso8583(raw_bytes: bytes) -> dict:
    # 1. Extract MTI (first 4 chars ASCII or 2 bytes binary)
    mti = raw_bytes[:4].decode('ascii')

    # 2. Extract bitmap (next 8/16 bytes)
    primary_bitmap = raw_bytes[4:12]
    has_extended = bool(primary_bitmap[0] & 0x80)

    # 3. Decode bitmap → which DEs are present
    present_des = []
    for byte_idx, byte in enumerate(primary_bitmap):
        for bit in range(8):
            if byte & (0x80 >> bit):
                present_des.append(byte_idx * 8 + bit + 1)
```

## Message Validation Checklist

1. **MTI**: Must be 4 digits, known type
2. **Bitmap**: Bit 1 of byte 1 = extended flag
3. **Bitmap parity**: Even number of 1s expected? (some implementations)
4. **DE presence**: Check required fields for MTI type
5. **DE length**: Fixed vs variable encoding
6. **Numeric fields**: All digits, no spaces for fixed
7. **PAN Luhn**: Validate DE002
8. **Expiry**: YYMM format, must be future
9. **Amount**: No negative, within limits
10. **MAC**: DE064 verified against HSM
