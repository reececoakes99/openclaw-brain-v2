# ISO20022 MX & SWIFT MT Reference

ISO20022 (MX messages) is the global financial messaging standard for SEPA, cross-border payments, and corporate banking. SWIFT MT (ISO15022) is the legacy scheme still active in correspondent banking. Neopay bridges both.

## ISO20022 MX Message Structure

Every MX message follows this envelope:

```
Swift message envelope (MX format)
└── AppHdr (optional header)
    ├── Sender (BIC8)
    ├── Receiver (BIC8)
    └── MsgIdentifier (msgId + seqNo)
└── Document
    └── {MessageType}
        ├── GrpHdr (group header — mandatory in all messages)
        └── {Body blocks specific to message type}
```

### Universal Block 1 — Message Header

```
{1:F01{BIC8}{BIC8}{NNNNNN}{NNNNNN}}{4:
 :20:{TxnRef}
 :21:{RltdRef}
 :25:{Account}
 :28E:{Page/Split/Seq}
 :50F:{Ordering Customer}
 :52A:{Ordering Institution}
 :53A:{Sending Institution}
 :54A:{Beneficiary Institution}
 :57A:{Account Institution}
 :58A:{Beneficiary Customer}
 :59F:{Beneficiary Customer name+addr}
 :70:{Remittance Info}
 :72:{Bank to Bank Info}
}
```

### Universal Block 4 — Message Body

Contains the business content, structured per message type.

### Universal Block 5 — Trailer

```
{5:{MAC}}{S:}{CHK:{checksum}}
```

## Key Message Types

### pain.001 — Customer Credit Transfer Initiation

Used to initiate SEPA/SEPA Instant credit transfers.

```xml
<Document xmlns="urn:iso:std:iso:20022:tech:xsd:pain.001.001.03">
  <CstmrCdtTrfInitn>
    <GrpHdr>
      <MsgId>{unique-msg-id}</MsgId>
      <CreDtTm>{YYYY-MM-DDTHH:MM:SS}</CreDtTm>
      <NbOfTxs>{count}</NbOfTxs>
      <SttlmInf>
        <SttlmMtd>{CLRG|SETA}</SttlmMtd>
        <SttlmAcct>
          <Id><Othr><Id>{iban}</Id></Othr></Id>
        </SttlmAcct>
      </SttlmInf>
    </GrpHdr>
    <PmtInf>
      <PmtInfId>{payment-info-id}</PmtInfId>
      <PmtMtd>{TRX|CLRG}</PmtMtd>
      <BkTxCd><Prtry><Cd>{proprietary-code}</Cd></Prtry></BkTxCd>
      <ReqdExctnDt>{YYYY-MM-DD}</ReqdExctnDt>
      <Dbtr>
        <Nm>{debtor-name}</Nm>
        <PstlAdr>...</PstlAdr>
      </Dbtr>
      <DbtrAcct><Id><IBAN>{iban}</IBAN></Id></DbtrAcct>
      <DbtrAgt><FinInstnId><BIC>{debtor-agent-bic}</BIC></FinInstnId></DbtrAgt>
      <CdtTrfTxInf>
        <PmtId><InstrId>{instr-id}</InstrId><EndToEndId>{e2e-id}</EndToEndId></PmtId>
        <Amt><InstdAmt Ccy="EUR">{amount}</InstdAmt></Amt>
        <CdtrAgt><FinInstnId><BIC>{creditor-agent-bic}</BIC></FinInstnId></CdtrAgt>
        <Cdtr>
          <Nm>{creditor-name}</Nm>
          <PstlAdr>...</PstlAdr>
        </Cdtr>
        <CdtrAcct><Id><IBAN>{iban}</IBAN></Id></CdtrAcct>
        <RmtInf><Ustrd>{remittance-info}</Ustrd></RmtInf>
      </CdtTrfTxInf>
    </PmtInf>
  </CstmrCdtTrfInitn>
</Document>
```

### pacs.008 — Financial Institution Credit Transfer

SEPA SCT/single credit transfer — used by payment service providers.

```xml
<Document xmlns="urn:iso:std:iso:20022:tech:xsd:pacs.008.001.02">
  <FiToFiCdtTrf>
    <GrpHdr>
      <MsgId>...</MsgId>
      <CreDtTm>...</CreDtTm>
      <NbOfTxs>1</NbOfTxs>
      <SttlmInf><SttlmMtd>CLRG</SttlmMtd></SttlmInf>
    </GrpHdr>
    <CdtTrfTxInf>
      <PmtId>
        <InstrId>...</InstrId>
        <EndToEndId>NOTPROVIDED</EndToEndId>
      </PmtId>
      <Amt><InstdAmt Ccy="EUR">100.00</InstdAmt></Amt>
      <CdtrAgt><FinInstnId><BIC>...</BIC></FinInstnId></CdtrAgt>
      <Cdtr><Nm>...</Nm></Cdtr>
      <CdtrAcct><Id><IBAN>...</IBAN></Id></CdtrAcct>
    </CdtTrfTxInf>
  </FiToFiCdtTrf>

```

## SWIFT MT (ISO15022) — Legacy Standard

SWIFT MT messages are structured differently from MX. They're field-based, colon-delimited.

### MT103 — Single Customer Credit Transfer

```
{1:F01{BICCODEA}{BICCODEB}{RUNNUMBER}{SEQUENCE}}
{2:I103{BICCODEB}{BICCODEA}{PACKETINFO}}{4:
 :20:{REFERENCE}
 :23B:{CREDITOR-CODE-TYPE}
 :23E:{INSTRUCTION-CODE}
 :26T:{TRANSACTION-TYPE-CODE}
 :30:{VALUE-DATE}
 :31C:{DATE-OF-ORIGINAL}
 :32A:{CURRENCY-CODE}{AMOUNT}
 :33B:{CURRENCY-CODE}{AMOUNT}
 :50A:{ORDERING-CUSTOMER-1}
   /50F:{ORDERING-CUSTOMER-2}
 :52A:{ORDERING-INSTITUTION-1}
   /52D:{ORDERING-INSTITUTION-2}
 :53A:{SENDING-RECEIVING-INSTITUTION-1}
 :54A:{RECEIVER-CORRESPONDENT}
 :56A:{INTERMEDIARY}
 :57A:{ACCOUNT-WITH-INSTITUTION}
 :58A:{BENEFICIARY-CUSTOMER-1}
   /58D:{BENEFICIARY-CUSTOMER-2}
 :59A:{BENEFICIARY}
   /59F:{BENEFICIARY}
 :70:{REMITTANCE-INFO}
 :71A:{CHARGES-SETTLEMENT}
 :72:{BANK-TO-BANK-INFO}
 :73:{CLEARING-INFO}
 :77B:{REGULATORY-INFO}
 :77T:{ENCODED-PARTY-INFO}
}{5:{MAC}}{S:}{CHK:{CHECKSUM}}
```

### SWIFT Field Types

| Type | Format | Example |
|------|--------|---------|
| :A: | BIC code | `BARCGB22` |
| :B: | BIC + account | `/12345678` |
| :C: | Account only | `/12345678` |
| :D: | Free text | unstructured text |
| :F: | Structured + code | `{addr}...` |
| :K: | 4-line address | multi-line |

## MT-to-MX Conversion Mapping

```
MT103  → pain.001.001.03 (customer credit transfer)
  :20  → GrpHdr/MsgId
  :50A/F → PmtInf/Dbtr (debtor)
  :59A/F → PmtInf/CdtTrfTxInf/Cdtr (creditor)
  :32A  → CdtTrfTxInf/Amt/InstdAmt (amount + ccy)
  :33B  → AdditionalAmt if present
  :70   → RmtInf/Ustrd
  :71A  → ChrgsAgt

MT202  → pacs.008.001.02 (FI credit transfer)
  :52A  → GrpHdr/InitgPty or DbtrAgt
  :58A/D → Cdtr/CdtrAcct
  :32A  → Amt/InstdAmt
  :33B  → ChrgsAmt
```

## SEPA Instant Credit Transfer (SCT Inst)

SEPA Instant requires:
- `MsgId` must be unique (no reuse within 20 days)
- `EndToEndId` cannot be `NOTPROVIDED` — must be `STRD` or custom
- Currency must be EUR in SEPA countries
- Maximum amount: €100,000 (configurable by scheme)
- Settlement: real-time via EBA CLEARING ( STEP2, RT1, EquensWorldline)

## Conversion Rules for POS Switch

When converting ISO8583 (HISO93) to SEPA MX:
```
Field 3 (ProcCode) → pain.001/PmtInf/BkTxCd/Prtry/Cd
Field 4 (Amount)   → CdtTrfTxInf/Amt/InstdAmt
Field 7 (DateTime) → GrpHdr/CreDtTm
Field 11 (STAN)     → PmtId/InstrId
Field 37 (RRN)      → PmtId/EndToEndId
Field 49 (Ccy)      → CdtTrfTxInf/Amt/@Ccy
Field 50           → Dbtr (name/IBAN)
Field 102          → CdtrAcct/IBAN
```
